Why Backends Install GitHub

Personal assistant gateway.

Text your assistant. push routes the message to Claude Code or Codex, keeps your memory in plain files, and sends the answer back over iMessage.

agent = "codex"
memory = "assistant/User.md"
state = "~/.push/state.json"

Own the assistant layer, not the runtime.

Gateway first

push owns messages, allowlists, routing, assistant config, memory, and conversation state.

Runtime disposable

Claude Code and Codex do the tool work. Future agent runtimes can be adapters, not rewrites.

Memory you can read

User profile and durable memory live in markdown, so you can inspect, edit, and version them.

How a message moves

Text yourself
push filters
Memory loads
Agent runs
Reply sends

Claude Code

Uses claude -p, persistent sessions, native tools, MCP, permissions, login, and CLAUDE.md behavior.

Codex

Uses codex exec, captures the Codex thread id, and resumes later turns with codex exec resume.

Install

Download a release archive from GitHub, copy config.example.json to config.json, set your iMessage handles, and choose claude or codex. 

cargo build --release
./target/release/push

# or download from releases
tar -xzf push-v0.1.0-*.tar.gz
push is MIT licensed and runs locally on your Mac.
