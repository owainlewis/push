The gateway is the product.

Everyone is building agent runtimes. push does not try to beat them. It gives those runtimes a durable personal-assistant layer.

Do not rebuild the agent

Claude Code, Codex, Cursor, and AMP are already competing on tools, permissions, models, shells, MCP, and repo context.

Own what should survive

Your assistant identity, memory, message history, allowlist, routing rules, and backend sessions belong in the gateway.

Why it matters

You get a personal assistant without betting the whole product on one agent runtime.

Use the agents you trust

Let Claude Code or Codex keep doing the hard execution work. Swap runtimes when one gets better.

Keep memory in plain files

Assistant profile and memory live in markdown you can inspect, correct, back up, and version.

Stay local

push polls Messages locally, shells out to local CLIs, and sends replies through Messages.

How a text becomes work

01 You send an iMessage
02 push checks the allowlist
03 Memory and profile load
04 Claude Code or Codex runs
05 The reply is sent back

Two runtimes today. More later.

The adapter contract is intentionally small: message in, assistant context in, backend session in, final reply out.

Claude Code

Uses claude -p, persistent sessions, native tools, permissions, MCP, login, and CLAUDE.md.

Codex

Uses codex exec, stores the Codex thread id, and resumes later turns with codex exec resume.

Next adapters

Other coding agents can plug in behind the same gateway without changing the assistant state model.

Install and text yourself

Install the binary, copy the example config, set your own iMessage handles, choose claude or codex, then run push on your Mac.

  • ✓macOS with Messages signed in
  • ✓Full Disk Access for your terminal
  • ✓Claude Code or Codex installed locally
curl -fsSL https://raw.githubusercontent.com/owainlewis/push/main/install.sh | sh

cp config.example.json config.json
# edit config.json

push --config config.json
